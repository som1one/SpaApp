```xml

    <string>UIInterfaceOrientationPortrait</string>
    <string>UIInterfaceOrientationPortraitUpsideDown</string>
```

## 📸 Решение проблемы 2: Правильные скриншоты

### Требования Apple:

1. **Скриншоты должны показывать реальное использование приложения**
2. **Не использовать маркетинговые материалы или placeholder'ы**
3. **Большинство скриншотов должны показывать основные функции**
4. **Скриншоты должны соответствовать реальному виду приложения**

### Размеры скриншотов:

#### iPhone 6.5" (iPhone 14 Pro Max, 13 Pro Max, 12 Pro Max)
- **Размер:** 1284 x 2778 пикселей
- **Формат:** PNG или JPEG
- **Минимум:** 1 скриншот, максимум: 10 скриншотов

#### iPad Pro 13" (iPad Pro 12.9")
- **Размер:** 2048 x 2732 пикселей
- **Формат:** PNG или JPEG
- **Минимум:** 1 скриншот, максимум: 10 скриншотов

## 📋 Пошаговая инструкция по созданию скриншотов

### Шаг 1: Подготовка симуляторов

#### Для iPhone 6.5":
1. Откройте Xcode
2. Запустите симулятор: **iPhone 14 Pro Max** или **iPhone 13 Pro Max**
3. Или используйте Flutter:
   ```bash
   flutter run -d "iPhone 14 Pro Max"
   ```

#### Для iPad Pro 13":
1. Откройте Xcode
2. Запустите симулятор: **iPad Pro (12.9-inch) (6th generation)**
3. Или используйте Flutter:
   ```bash
   flutter run -d "iPad Pro (12.9-inch) (6th generation)"
   ```

### Шаг 2: Выбор экранов для скриншотов

**Рекомендуемые экраны (в порядке приоритета):**

1. ✅ **Главный экран (Home)** - показывает основные функции
2. ✅ **Экран услуг/меню** - демонстрирует каталог
3. ✅ **Экран записи/бронирования** - показывает процесс записи
4. ✅ **Экран профиля** - показывает личный кабинет
5. ✅ **Экран бонусов/лояльности** - показывает программу лояльности
6. ✅ **Экран истории записей** - показывает список записей

**НЕ используйте:**
- ❌ Splash screen (экран загрузки)
- ❌ Экран входа/регистрации (только если это не основная функция)
- ❌ Placeholder'ы или маркетинговые материалы
- ❌ Пустые экраны

### Шаг 3: Создание скриншотов

#### На Mac (Xcode Simulator):

1. **Запустите приложение в симуляторе**
2. **Перейдите на нужный экран**
3. **Сделайте скриншот:**
   - Нажмите `⌘ + S` (Command + S)
   - Или: `Device → Screenshot` в меню симулятора
4. **Скриншот сохранится на Рабочий стол**

#### На Windows (через Flutter):

1. **Запустите приложение:**
   ```bash
   flutter run -d "iPhone 14 Pro Max"
   ```
2. **Используйте инструменты разработчика браузера** (если запущено в Chrome)
3. **Или используйте эмулятор Android Studio** с настройкой размера экрана

#### Альтернативный способ (рекомендуется):

**Используйте реальное устройство через TestFlight:**
1. Установите приложение на iPhone/iPad через TestFlight
2. Сделайте скриншот на устройстве (кнопки питания + громкость)
3. Откройте Фото на Mac
4. Экспортируйте в нужном размере

### Шаг 4: Редактирование скриншотов (если нужно)

**Требования:**
- ✅ Размер должен быть точно: **1284 x 2778** (iPhone) или **2048 x 2732** (iPad)
- ✅ Формат: PNG (предпочтительно) или JPEG
- ✅ Не добавляйте рамки устройства (Apple добавит автоматически)
- ✅ Не добавляйте текст или графику поверх скриншота

**Инструменты для редактирования:**
- **Mac:** Preview (встроенный), Pixelmator, Photoshop
- **Windows:** Paint.NET, GIMP, Photoshop
- **Онлайн:** Photopea.com (бесплатный аналог Photoshop)

### Шаг 5: Загрузка в App Store Connect

1. **Войдите в App Store Connect:**
   - https://appstoreconnect.apple.com
   - Выберите ваше приложение
   - Перейдите в **App Store → Версия для iOS**

2. **Найдите раздел "Снимки экрана":**
   - Прокрутите до раздела **"Снимки экрана и предпросмотр"**
   - Нажмите **"View All Sizes in Media Manager"** (если нужно)

3. **Загрузите скриншоты:**
   - **iPhone 6.5":** Перетащите файлы в область "iPhone 6.5" Display"
   - **iPad Pro 13":** Перетащите файлы в область "iPad Pro (13-inch) Display"
   - Загрузите минимум 1 скриншот для каждого размера

4. **Проверьте порядок:**
   - Первый скриншот должен показывать главный экран
   - Остальные - основные функции приложения
