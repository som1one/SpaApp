## 🧪 Как протестировать

### Вариант 1: Симулятор iOS (быстро)

**Плюсы:**
- Быстро
- Не нужен реальный iPhone
- Можно тестировать разные размеры экранов

**Минусы:**
- Не все функции работают (камера, геолокация, push)
- Может отличаться от реального устройства

**Как запустить:**
```bash
flutter run -d "iPhone 14 Pro Max"
```

### Вариант 2: Реальный iPhone (рекомендуется)

**Плюсы:**
- Реальное поведение
- Все функции работают
- Можно проверить производительность

**Минусы:**
- Нужен Mac и iPhone
- Или используйте TestFlight

**Как запустить:**
1. Подключите iPhone к Mac
2. Разрешите разработку на iPhone
3. Запустите: `flutter run`

### Вариант 3: TestFlight (лучший вариант)

**Плюсы:**
- Тестирование на реальном устройстве
- Не нужен Mac
- Можно дать другим людям

**Минусы:**
- Нужно собрать и загрузить в TestFlight
- Занимает время

**Как использовать:**
1. Соберите через Codemagic
2. Загрузите в TestFlight (уже сделано)
3. Установите через TestFlight на iPhone
4. Тестируйте

## 🆘 Типичные проблемы iOS

### Проблема 1: "App crashes on launch"

**Причина:**
- Неправильные разрешения в `Info.plist`
- Отсутствующие зависимости
- Проблемы с подписью

**Решение:**
- Проверьте логи в Xcode
- Проверьте `Info.plist`
- Убедитесь, что все зависимости установлены

### Проблема 2: "White screen"

**Причина:**
- Ошибка в коде
- Проблемы с WebView
- Неправильная инициализация

**Решение:**
- Проверьте логи
- Запустите в debug режиме
- Проверьте консоль в Xcode

### Проблема 3: "Keyboard covers input"

**Причина:**
- Неправильная настройка `Scaffold`
- Отсутствие `resizeToAvoidBottomInset`

**Решение:**
```dart
Scaffold(
  resizeToAvoidBottomInset: true,
  // ...
)
```

### Проблема 4: "Permissions not working"

**Причина:**
- Не указаны в `Info.plist`
- Неправильный запрос разрешений

**Решение:**
- Добавьте в `ios/Runner/Info.plist`:
```xml
<key>NSCameraUsageDescription</key>
<string>We need camera access</string>
```
